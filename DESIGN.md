# 墨·境 — 设计文档（Python 实现版）

> Python 实现版 | 2026-07-08
>
> 太极拳驱动的三维粒子宇宙。手为引力源，慢为钥匙，稳为修行。
> 几千个墨色粒子漂浮在深空中，你的动作形成力场，粒子如活物般被吸引、环绕、绽放。
>
> 本版不采用网页/HTML/Three.js 方案，改为 Python 桌面应用：用 Python 编写粒子物理、姿态输入、声音合成与 OpenGL 渲染。

## 一、概念总纲

### 一句话

**你站进一片深空，几千个粒子在沉睡。你慢下来，它们醒来，飘向你。**

### 核心体验

一面大屏幕（电视或投影），画面是一片深邃的靛蓝黑空间。空间里漂浮着数千个墨色粒子——不是平面上的墨点，而是在三维空间中悬浮、漂移、呼吸的微小光体。

你站在屏幕前打太极拳。摄像头追踪你的身体，你的手在物理空间中移动，屏幕里的粒子宇宙随之响应：

- **快速挥手**：粒子被你推开，手所到之处粒子向两侧散开，像手指划过水面。什么都不会留下。
- **开始慢下来**：粒子不再逃跑。它们开始从四面八方飘向你的手，在你指尖汇聚、旋转。
- **匀速、稳定地慢行**：粒子不仅跟着你，还围绕你的运动轨迹形成漩涡、螺旋、环流——像微型的银河悬臂在你的手臂周围旋转。
- **站桩静止**：整个空间的粒子向你涌来。你的身体轮廓被粒子包裹，变成一个发光体。静止3秒，粒子向外爆发再缓缓回落——像一次深长的呼吸。

### 体验金句

> 快，是闯入者。慢，是归人。静止，是你成为了这个宇宙的中心。

### 与传统"太极+视觉"类作品的根本差异

大多数同类作品的做法：识别招式 → 触发对应动画/特效。这是 **"按键-响应"模式**，只不过把键盘换成了身体姿势。

本系统的做法：**力场模式。** 没有招式识别，没有预设动画。你的运动质量（速度、稳定性、持续性）在空间中产生一个动态力场，粒子在这个力场中自主运动。系统有生命，你不是"操作"它，你是"影响"它。

## 二、空间设计

### 2.1 空间的面貌

一个三维的粒子宇宙。不是星空，不是海底，介于两者之间——有星云的深邃，也有深海的沉静。

**纵深层次：**
- 画面不是平的。近处的粒子大而亮（距离屏幕面近），远处的粒子小而暗（深入屏幕后方）
- 整个粒子体积是一个约 10×6×8 米（世界坐标）的长方体空间
- 摄像头画面映射到空间的前表面（z=0 平面），空间向屏幕"里面"延伸

**空间中的元素：**

| 元素 | 数量 | 行为 |
|------|------|------|
| **暗粒子**（主体） | 5000-8000 | 缓慢布朗运动，受手部力场影响。不发光，仅微弱散射环境光 |
| **明粒子**（激活态） | 0-2000 | 被手吸引/捕获的暗粒子转化为明粒子。亮度提高，颜色偏冷白 |
| **金粒子**（GA产物） | 0-300 | GA进入高阶后从明粒子中产生。暖金色，缓慢上升 |
| **轨迹余晖** | 持续生成 | 手移动时在身后留下的粒子带。亮度高，3-5秒后消散 |

### 2.2 用户身体的空间映射

MediaPipe 提供 33 个 3D 关键点。系统使用其中 5 个核心点：

| 关键点 | 用途 |
|--------|------|
| 右手腕 | 主力场源。吸引力/排斥力的中心 |
| 左手腕 | 辅助力场源（强度为右手腕的 60%） |
| 右肩 + 左肩 | 计算肩宽，归一化所有速度 |
| 鼻尖（头部中心） | 身体中轴参考，用于粒子对身体轮廓的感知 |

每个关键点携带：3D位置 (x, y, z)、从帧间差分计算的速度、EMA平滑速度、稳定性评分。

### 2.3 力场源模型

手不只是空间中的一个点。它是一个"力场源"——有自己的影响半径、强度、力类型。

**右手腕力场的空间结构（球对称，但强度在径向上渐变）：**

```
半径 r（世界单位，1单位 ≈ 屏幕宽度的1/5）

r < 0.3   "核心区" — 粒子直接吸附到手上，与手一起移动
r 0.3-1.0 "近场区" — 力场主导，粒子被强烈吸引/排斥
r 1.0-3.0 "中场区" — 力场减弱，粒子有响应但保留自主漂移
r 3.0-6.0 "远场区" — 微弱的吸引力（站桩时扩大到此范围）
r > 6.0    "自由区" — 粒子不受影响，自由布朗运动
```

影响半径不是固定的——它随运动状态动态变化：
- 快速移动 → 影响半径缩小（粒子来不及反应，只有近处的被推开）
- 慢速移动 → 影响半径扩大（粒子有足够时间感知并飘过来）
- 站桩 → 影响半径扩到最大，整个空间的粒子都在"感知"你

## 三、粒子物理引擎

### 3.1 单个粒子的状态

```javascript
Particle {
    position: Vector3,      // 世界坐标
    velocity: Vector3,      // 当前速度
    mass: float,            // 0.5-2.0，随机，影响惯性
    baseColor: Color,       // 粒子的基础颜色（墨色系，每个粒子微调）
    brightness: float,      // 0-1，当前亮度（0=休眠暗粒子，1=完全激活）
    size: float,            // 当前渲染大小
    life: float,            // 0-1，粒子生命阶段（新生→成熟→衰老）
    activated: boolean,     // 是否被手激活
    activationTime: float,  // 被激活的持续时间（秒）
    trailTimer: float,      // 轨迹粒子专属：剩余寿命
}
```

### 3.2 每帧粒子更新

每个粒子每帧经历以下力的叠加：

```
合力 = 手部力场 + 布朗漂移 + 粒子间微弱吸引 + 拖拽力 + 边界约束
```

#### 力1：手部力场（核心）

由手腕位置、速度、稳定性共同决定。这是整个系统最重要的力。

```
force_hand = 0

if distance_to_hand < influence_radius:

    if emaSpeed > 1.0:  // 快速——排斥
        strength = repulsionStrength * (1 - distance/influence_radius)
        direction = (particle - hand).normalized()
        force_hand += direction * strength
    
    elif emaSpeed > 0.15:  // 中速——弱排斥/中性
        strength = weakRepulsion * (1 - distance/influence_radius)
        direction = (particle - hand).normalized()
        force_hand += direction * strength * 0.3  // 弱化
    
    elif emaSpeed > 0.03:  // 慢速——吸引力
        strength = attractionStrength * (1 - distance/influence_radius)
        direction = (hand - particle).normalized()
        force_hand += direction * strength
        // 额外：切线力产生旋转
        tangential = cross(direction, handVelocity).normalized()
        force_hand += tangential * vortexStrength * stability
    
    elif emaSpeed > 0.008:  // 极慢——强吸引 + 强漩涡
        strength = strongAttraction * (1 - distance/influence_radius)
        direction = (hand - particle).normalized()
        force_hand += direction * strength
        tangential = cross(direction, handVelocity).normalized()
        force_hand += tangential * vortexStrength * stability * 2.0
    
    else:  // 静止——长程强吸引
        strength = convergenceStrength * (1 - distance/maxInfluenceRadius)
        direction = (hand - particle).normalized()
        force_hand += direction * strength
```

**关键：** 力场强度在手速的每个区间是**连续函数**，不是跳变。emaSpeed 在两个区间边界附近，力场类型平滑过渡。

#### 力2：布朗漂移

每个粒子有独立的 Perlin 噪声驱动的缓慢漂移。让粒子在不受手影响时看起来"活着"而不是"死了"。

```
drift = perlin3D(particle.position + time) * driftAmplitude
```

每个粒子使用不同的噪声种子，且噪声时间尺度极慢（周期约30-60秒），所以漂移看起来是缓慢的、有机的、非重复的。

#### 力3：粒子间微弱吸引

模拟粒子云的自组织倾向。仅在粒子密度较高的区域生效。

```
for each nearby particle within 0.5 units:
    force += (other.position - particle.position) * 0.02 * (1 - distance/0.5)
```

这个力很弱——约为手部力场的 1-5%。它让靠近手的粒子群有一定凝聚力，不会散成乱沙。

#### 力4：拖拽力

模拟空间介质的阻尼。粒子不会无限加速。

```
force_drag = -particle.velocity * dragCoefficient
```

dragCoefficient 在 0.3-0.8 之间，粒子质量越大拖拽越小。产生"惯性感"——大粒子比小粒子更难改变方向。

#### 力5：边界约束

粒子不应该飘出可视空间。

```
if particle outside volume:
    soft force pushing it back in
```

不是硬边界反弹，是柔和的回推力。粒子可以短暂超出边界但会被慢慢推回。

### 3.3 轨迹余晖

手每移动一定距离（累积位移触发，同旧方案），在手的当前位置生成一批"轨迹粒子"：

```
轨迹粒子属性：
- 位置：手当前位置 + 微小随机偏移（0-0.2单位）
- 速度：手当前速度 × 0.3 + 随机微漂移
- 颜色：由当前墨色层级决定（快=淡清墨色，慢=浓焦墨色）
- 亮度：0.7-1.0（比空间暗粒子亮很多）
- 生命：3-5秒，线性衰减
- 大小：1-3单位，随生命衰减
- 不受后续力场影响（轨迹粒子一旦生成就"冻结"，只漂移+衰减）
```

轨迹粒子在空间中连成线。不是"画"出来的二维墨迹——是漂浮在三维空间中的发光余晖带。手快速滑过，留下的轨迹像彗星的尾巴；手慢慢移动，留下的轨迹像银河的尘埃带。

### 3.4 暗粒子↔明粒子的转化

- 暗粒子进入手部力场核心区（r < 0.3）且 emaSpeed < 0.15 → 逐渐点亮（brightness 平滑增加，每秒+0.5）
- 明粒子离开手部力场 → 逐渐变暗（brightness 平滑衰减，每秒-0.3）
- 转化不是瞬时的，有约1-2秒的过渡。这保证了手移动时，身后留下一道"光迹"——不是因为新粒子被生成，而是因为刚被点亮的粒子还在慢慢暗下去

## 四、运动→力场映射（EMA速度 + 稳定性 → 力场参数）

### 4.1 映射总表

| 层级 | emaSpeed (肩宽/s) | 力场类型 | 影响半径 | 视觉表现 |
|------|-------------------|---------|---------|---------|
| 0·排斥 | > 1.0 | 强排斥 | 1.5 单位 | 粒子被推开，手周围形成空洞 |
| 1a·逃逸 | 0.15-1.0 | 弱排斥 | 2.0 单位 | 粒子微微避开手，但不剧烈 |
| 1b·吸引 | 0.03-0.15 | 吸引力 + 弱漩涡 | 3.5 单位 | 粒子开始向手汇聚，身后形成粒子尾迹 |
| 2·环绕 | 0.008-0.03 | 强吸引 + 强漩涡 | 5.0 单位 | 粒子绕手旋转，形成螺旋臂，轨迹余晖浓密 |
| 3·归一 | ≤ 0.008 | 长程强吸引 | 8.0 单位 | 全空间粒子向身体汇聚，静止3秒触发绽放 |

### 4.2 稳定性的调制作用

稳定性（2秒窗口内的速度方差）不改变力场类型，但改变力场的"品质"：

- **高稳定性 (>0.85)：** 力场平滑、干净。粒子运动轨迹流畅，漩涡对称。暗粒子点亮后颜色偏冷白（干净的光）。
- **中稳定性 (0.5-0.85)：** 力场有轻微抖动。粒子运动带微小的不规则扰动。正常水平。
- **低稳定性 (<0.5)：** 力场抖动明显。粒子在吸引和排斥之间反复横跳（因为速度不稳定）。漩涡破碎。粒子点亮后颜色偏暖浊（浑浊的光）。

### 4.3 迟滞切换（同旧方案，适配3D力场）

- 升级（更慢→更好）：emaSpeed 连续 3 帧低于目标阈值 → 力场类型切换
- 降级（更快→更差）：emaSpeed 连续 8 帧高于目标阈值 → 力场类型切换

力场参数（排斥力大小、吸引力大小、影响半径）在切换时平滑过渡，过渡时间约0.3秒。

## 五、视觉系统

### 5.1 色彩体系

**空间底色：** 不是纯黑。是极暗的靛蓝。

```
深空底色: #080812 (RGB 8, 8, 18)
这个底色在屏幕上看起来几乎是黑的，但带有极微弱的蓝紫倾向。
不是"关了灯"，是"在宇宙深处"。
```

**暗粒子（休眠态）：**

粒子不是灰色。每个粒子有微弱的色调偏移——大部分偏向蓝紫，少部分偏向暖褐。这让粒子云在休眠时不是"一团灰"，而是有微妙的色彩变化的"雾"。

```
暗粒子色域 (HSL):
  色相: 240°-280°（蓝→紫），约80%的粒子
        20°-40°（暖褐），约20%的粒子
  饱和度: 8-25%
  亮度: 8-18%
```

单个粒子在色域中随机取值。整体看起来是"墨色中透出微弱的蓝紫冷光和极少数暖色微粒"。

**明粒子（激活态）：**

被手激活的粒子亮度提升，饱和降低，色相向冷白偏移。

```
激活色域:
  brightness 0.0 → 暗粒子本色
  brightness 0.5 → 色相不变，饱和度减半，亮度翻倍
  brightness 1.0 → 色相向220°（蓝白）偏移，饱和度趋近5%，亮度60-80%
```

激活粒子看起来是"冷白带一点原来的色调"——蓝紫粒子变成蓝白，暖褐粒子变成暖白。

**轨迹粒子：**

轨迹粒子的颜色由手当前的 emaSpeed 映射到墨色五级（保留旧方案的墨色系统）：

```
emaSpeed 0.008 → 焦墨: rgba(10, 5, 0) → RGB暖黑，高不透明度
emaSpeed 0.03  → 浓墨: rgba(18, 10, 3)
emaSpeed 0.06  → 重墨: rgba(30, 20, 10)
emaSpeed 0.15  → 淡墨: rgba(45, 35, 25)
emaSpeed 1.0   → 清墨: rgba(60, 50, 40) → RGB暖灰，低不透明度
emaSpeed > 1.0  → 无轨迹粒子生成
```

轨迹粒子在空间中形成暖墨色的"彗尾"——与冷色的激活粒子形成对比。墨尾是暖的，粒子是冷的。

**金粒子（GA产物，见第六章）：**

```
金色域 (HSL):
  色相: 38°-48°
  饱和度: 40-70%
  亮度: 40-80%（随GA升高）
```

### 5.2 粒子渲染

**核心方案：** Python + Pygame 窗口 + ModernGL/OpenGL 点精灵 + NumPy 粒子数组 + 可选 Bloom 后处理。

**单粒子形状：** 在片元着色器中用 `gl_PointCoord` 程序生成柔软圆形，不需要外部图片：
- 中心 100% 不透明，纯白（最终颜色由 OpenGL 顶点颜色决定）
- 半径 0-40%：保持 100% 不透明
- 半径 40-80%：从100%线性衰减到0%不透明
- 半径 80-100%：完全透明
- 边缘带 2px 的极微弱高斯模糊

这个着色器让每个粒子拥有柔软的圆形外观。Python 端只上传位置、颜色、大小和亮度数组，GPU 端完成点精灵展开、透明度衰减和加法混合。

**顶点颜色：** 每个粒子的颜色存储在 OpenGL VBO 的 color attribute 中。每帧由 NumPy 批量更新 position / color / size，再通过 `buffer.write()` 上传到 GPU。这样每个粒子可以有独立颜色、亮度和尺寸，同时避免 Python 层逐个绘制。

**粒子大小：**
- 暗粒子：2-6 世界单位（远处小，近处大）
- 明粒子：暗粒子的 1.5-2 倍
- 轨迹粒子：4-10 世界单位（比粒子大，因为它们要形成可见的"线"）
- 金粒子：2-5 世界单位

**Bloom 后处理（可选增强）：**

使用离屏 Framebuffer + 两次高斯模糊 + 加法合成：
- bloom强度：低，约 0.3-0.5
- bloom半径：约 0.5
- bloom阈值：只让亮度 > 0.4 的粒子产生辉光（暗粒子不发光，明粒子和轨迹粒子发光）

Bloom 在 Python 版中作为可关闭增强项。默认先保证粒子运动和交互流畅；如果掉帧，则降低 Bloom 分辨率、关闭 Bloom，或把粒子数从 8000 降到 4000-6000。

**深度雾（fog）：**

在顶点/片元着色器中根据 z 深度计算指数雾，远处粒子被雾气柔化。雾的颜色和空间底色一致（#080812）。雾的密度设为：在 z=-6（空间最深处）处粒子透明度降到约 30%。

这创造了自然的深度感——远处的粒子不仅小，而且被"空间雾"柔化，像隐没在深海中。

### 5.3 空间构图与摄像机

**默认视角：** 透视摄像机，FOV 70°，位置略微俯视（摄像机位置 y=1.0，看向原点偏下）。

这个视角给观众"站在高处俯瞰粒子云"的感觉，同时保持足够的深度感。

**视角可切换（按 V 键）：**
- 默认视角：略俯视
- 正面视角：正对空间，更强的沉浸感
- 追踪视角：摄像机缓慢跟随手部力场中心移动（延迟跟随，约1秒滞后）

**空间的物理尺寸映射：**
- 屏幕宽度对应世界坐标 X: -5 到 5
- 屏幕高度对应世界坐标 Y: -3 到 3（中心在屏幕中央）
- 深度对应世界坐标 Z: 0 到 -8（0=屏幕面，-8=最深处）

### 5.4 两手腕双力场

左手腕也产生力场，但强度为右手腕的 60%。这允许双手推手动作产生两个力场中心：

- 两手靠近时：两个力场叠加，粒子汇聚更强烈
- 两手分开运动时：粒子在两个力场之间被拉扯，形成"粒子桥"
- 推手时（两手交替推动）：粒子在两个力场之间来回流动

这是双手天然产生的视觉效果，不需要额外的"推手模式"代码。

## 六、内力共鸣系统（金气值 GA）

### 6.1 GA 机制（保留旧方案核心逻辑）

```
if stability > 0.7:
    GA += 0.04 × dt    // +4%/秒, 25秒从0到1
else:
    GA -= 0.08 × dt    // -8%/秒, 12.5秒从1到0
GA ∈ [0, 1]
```

### 6.2 四阶表现（3D 适配版）

#### 第一阶 · 暖意 — GA 0.05-0.25

**视觉：** 激活粒子的色温微微变暖——冷白中出现极细微的暖调。不是颜色明显变了，是"感觉"变了。暗粒子不受影响。

**粒子行为：** 无变化。

**触发时间：** 约 1.3 秒稳定运动后。

#### 第二阶 · 微光 — GA 0.25-0.50

**视觉：** 手周围 r < 0.5 的激活粒子出现暖金色微晕。粒子本身仍是冷色的，但其边缘的辉光（bloom）从冷白渐变为暖金。

**粒子行为：** 在手部力场核心区（r < 0.3），新激活的粒子有小概率（~10%）携带暖金色调，而不是蓝白色调。

**声音变化：** 轨迹粒子生成时的声音从纯正弦波变成带有微弱二次谐波（八度音）。

**触发时间：** 约 6.3 秒稳定运动后。

#### 第三阶 · 浮金 — GA 0.50-0.75

**视觉：** 手周围出现专属的金粒子。金粒子从手的力场中"诞生"（从暗粒子转化而来），缓慢向上/向外飘升。

```
金粒子行为：
- 生成速率：每秒 2-5 个
- 生成位置：手部力场核心区 (r < 0.3)
- 速度：初始向上为主 (vy: 0.5-2.0 单位/s)，带随机水平漂移
- 不受手部力场影响（金粒子一旦生成就独立飞行）
- 生命：4-8秒，后2秒渐隐
- 上升过程中带缓慢的正弦水平振荡（波长 ~3s）
```

金粒子看起来像香炉里升起的烟缕中偶然闪过的火星，或篝火上浮起的余烬。

**粒子行为：** 手部力场影响范围内的粒子开始出现自发排列——不再是随机分布，而是倾向于沿力场线排列，形成隐约的"流线"。

**声音：** 每颗金粒子生成时有 30% 概率触发一个极轻的高音（五声音阶高音区，-22dB），像风铃偶然被微风吹响。

**触发时间：** 约 12.5 秒稳定运动后。

#### 第四阶 · 墨韵 — GA 0.75-1.0

**视觉——四个效果同时呈现：**

**4a. 粒子呼吸：** 每隔 4-6 秒，全空间的暗粒子同步发生一次微弱的亮度脉冲——整体亮 3-5%，持续约 1.5 秒，然后缓慢回落。像空间本身在呼吸。

**4b. 力场可视：** 手部力场的"边界"变得隐约可见——不是一条线，而是在影响半径边界处，粒子密度有一个隐约的环状峰值。像能量场的边缘被粒子勾勒出来。

**4c. 金粒子密度增加：** 金粒子生成速率翻倍（4-10个/秒），并且更大的金粒子（原本2-5单位→3-8单位）。部分金粒子在上升过程中分裂成2-3个更小的粒子。

**4d. 暗粒子暖化：** 距离手较远的暗粒子也开始微微暖化——整个空间从"深冬夜空"渐变到"深秋夜空"。变化极慢（约30秒完成过渡），不会被察觉但会被感知。

**声音：** 每次粒子呼吸脉冲时，响起极低沉的共鸣（80-100Hz正弦波 + 微弱三次谐波，attack 0.5s，release 2s，-20dB）。类似颂钵被轻敲后贴着耳朵听的感觉。金粒子声音频率翻倍。

**触发时间：** 约 19 秒稳定运动后。GA 满值（1.0）需 25 秒。维持第四阶需要持续高稳定性——一旦掉到 0.75 以下就退回第三阶。

### 6.3 GA 的视觉指示器

画面左下角，一个极小的、半透明的环。环的亮度/颜色随 GA 变化：
- GA 0.0：环几乎不可见（深灰，透明度 5%）
- GA 0.5：环呈暖铜色，透明度 20%
- GA 1.0：环呈暖金色，透明度 30%，带极微弱的外发光

不是 UI 条，是"一盏远处的灯"。用户余光能感知到火在烧还是快灭了。用户可以选择关掉它（按 H 键）。

## 七、静止绽放

### 7.1 触发条件

```
emaSpeed ≤ 0.016 肩宽/s（层3·归一）
且 stillnessFrames > 58（约1秒 @ 60fps）
且 bloomCooldown ≤ 0
```

### 7.2 绽放过程（三个阶段）

**第一阶段 · 汇聚（触发前约1秒稳定期间）：**

粒子已经开始向手部力场中心汇聚。这不是绽放的一部分，是静止状态下力场本身的吸引效果。这里的"慢"不是等待很久，而是稳定和蓄势：系统应在约1秒内感知到你已经稳住。

**第二阶段 · 爆发（触发瞬间，持续约 0.2-0.35 秒）：**

以手的力场中心为原点，所有 r < 2 的粒子获得向外的爆发速度。不是均匀的球形爆炸——爆发偏向空间的上半球和前方（z轴正方向，即屏幕外，向观众方向）。

```
爆发速度：
- 径向向外：速度 6-12 单位/s，随距离衰减
- 向上偏置：Y方向 +30%
- 向前偏置：Z方向 +20%（粒子向屏幕外的观众"涌来"）
- 带切向随机分量：形成花朵绽放而非炸弹爆炸的形状
```

爆发后约 0.15-0.25 秒，粒子速度的反转开始——向外速度快速衰减，向内的引力重新生效。

**第三阶段 · 回落（爆发后约 0.4-1.2 秒）：**

粒子回到手部力场中心。不是直线回落——它们在回落过程中带着残留的切向速度，形成短促的螺旋回收。太极的"慢"来自稳定和控制，不是把所有视觉过程都拖成慢镜头；绽放应当像一次干净的发劲。

### 7.3 绽放期间的视觉效果

- 所有参与爆发的粒子瞬间亮度提升到 1.0（全亮），颜色变为接近纯白
- 绽放粒子边缘出现暖金色辉光（无论当前 GA 等级）
- 生成 50-80 个绽放专属粒子：短寿命（1.5-3秒）、纯暖金色、从爆发中心辐射
- 已存在的轨迹余晖被爆发扰动——它们微微向外推了一下

### 7.4 冷却

绽放后 5 秒内不重复触发。冷却期间，即使静止条件满足，只维持层3的吸引状态，不触发第二次爆发。

### 7.5 设计意图

绽放是对太极最高境界——站桩的致敬。在太极中，"不动"比"动"更难、更见功力。站桩时体内发生的是复杂的微调——肌肉的松紧、呼吸的深浅、重心的微移。这些是观众看不见的。

绽放让"看不见的内在丰富性"变得可见。它不是系统凭空制造的效果，而是 3 秒静止的合乎逻辑的终点——粒子被吸引了 3 秒，汇聚到极限，然后"溢出来"。

## 八、声音设计

### 8.1 总体原则

- 所有声音实时合成，零预录素材
- 使用五声音阶（钢琴黑键 C, D, E, G, A），保证任意组合可听
- 音高由手部 Y 坐标映射（3 个八度，15 个音高）
- 使用 StereoPannerNode，手在画面左侧→声音偏左，反之偏右。pan 范围 ±0.4（含蓄的空间感）
- 总音量 -18dB，声音是墨迹的"影子"，不是主角

### 8.2 声音事件

| 事件 | 声音 | 实现 | 音量 |
|------|------|------|------|
| 轨迹粒子生成 | 极短纯音，频率=音高映射，attack 2ms, release 20-60ms（慢则长）。慢时叠加微弱二次谐波 | OscillatorNode(sine) + 可选 OscillatorNode(×2, 15%音量) | -18dB |
| 粒子被激活（暗→明） | 极微弱的"沙"声。白噪声通过带通滤波器，中心频率=音高映射/3，Q=1.5，持续约80ms | Noise → BiquadFilter(bandpass) → Gain(envelope) | -26dB |
| 漩涡形成（层2中 stability>0.85） | 低频持续共鸣。两个正弦波差1.5Hz → 缓慢拍频，基频80-150Hz随Y坐标变化 | 2×OscillatorNode(sine) → Gain(缓慢LFO调制, ±15%) | -24dB |
| 绽放爆发 | 基频=音高映射的6音泛音和弦（频率比1:2:3:4:5:6），同时响起，各自独立衰减(0.8-3s) | 6×OscillatorNode(sine) → 独立Gain(envelope) | -12dB |
| 绽放回落 | 低频缓慢共鸣，基频60Hz，带非常慢的振幅调制（≈0.3Hz），持续约3秒 | OscillatorNode(sine, 60Hz) → Gain(LFO, 0.3Hz) | -20dB |
| 金气二阶 | 轨迹粒子声音叠加高八度泛音（频率×2，音量15%） | 同轨迹粒子 + 第二个OscillatorNode | 泛音-30dB |
| 金气三阶 | 金粒子生成时30%概率触发随机高音（五声音阶A5-D6），attack 5ms, release 150-300ms | OscillatorNode(sine) → Gain(envelope) | -22dB |
| 金气四阶·粒子呼吸 | 颂钵共鸣。基频80-100Hz + 三次谐波，attack 0.5s, release 2s。 | OscillatorNode(sine) + OscillatorNode(×3, 8%音量) → Gain | -20dB |
| 快(层0) | 完全静默 | — | — |

### 8.3 声音密度管理

当力场中有大量粒子同时被激活时，声音事件会叠得非常密集。需要做限流：

- 轨迹粒子声音：每 50ms 内最多触发 3 个。超出排队的丢弃
- 激活声音：每 100ms 内最多触发 5 个
- 金粒子声音：本来就很低频（每秒0.6-1.5次），不需要限流

## 九、双手与推手

### 9.1 双手力场

左右手各自产生力场。右手为主力源（强度 1.0），左手为辅助力源（强度 0.6）。

正常打拳时，双手交替运动 → 两个力场在空间中交错。粒子在两个力场之间来回流动。

### 9.2 双手交汇

当两手距离 < 1.0 世界单位时：
- 两个力场叠加，中心的吸引力增强
- 粒子在两手腕之间形成高密度的"粒子桥"
- 双手缓慢分开时，粒子桥被拉伸，中间变细，最终断裂——粒子分流到两手各自跟随

### 9.3 推手暗示

两人推手不在 v1 范围内。但单人打拳时双手的交替推进，自然就会形成粒子在两个力场之间来回流动的效果。这个效果不需要特殊代码——双力场的自然叠加就能产生。

## 十、技术架构

### 10.1 整体栈

| 层 | 技术 | 理由 |
|----|------|------|
| 姿态估计 | `opencv-python` + `mediapipe` Python | 摄像头读取稳定，Python 内直接得到 33 个姿态点 |
| 交互兜底 | 鼠标/触控板双手模拟模式 | 没有摄像头或模型安装失败时，仍可立即检查粒子表现 |
| 3D渲染 | `pygame` + `moderngl` + GLSL | Pygame 负责窗口/输入，ModernGL 负责 GPU 点精灵渲染 |
| 粒子物理 | `numpy` 向量化数组 | Python 中避免逐对象更新，6000-8000 粒子仍可实时 |
| 后处理 | OpenGL Framebuffer + shader blur | Bloom 可开关，优先保证交互帧率 |
| 声音 | `pygame.mixer` + NumPy 实时生成短音频缓冲 | 不使用预录音乐，事件触发时即时合成音色 |
| 噪声 | Python 实现 value/simplex noise 或 NumPy 随机场 | 驱动粒子布朗漂移，保证有机感 |
| 部署 | Python 桌面应用 | 本地运行，便于调参、接摄像头和展览设备 |

## 十二、边界情况

| 场景 | 处理 |
|------|------|
| 用户离开摄像头 | 姿态丢失 → 冻结力场（保持最后一帧的影响），所有粒子缓慢回到自由漂移状态。不崩溃 |
| 重新进入摄像头 | 姿态恢复 → 力场从最后一帧平滑过渡到当前帧（0.5秒过渡），防止粒子"瞬移" |
| 低光环境 | MediaPipe关键点抖动 → EMA平滑抑制高频抖动。调试面板提示"低置信度"，力场轻微抖动但不会剧烈跳变 |
| 摄像头被占用/打不开 | 显示提示，自动切换到 `--input mouse` 演示模式。不崩溃 |
| MediaPipe未安装或初始化失败 | 终端输出安装建议，窗口内进入鼠标模式，仍可检查视觉核心 |
| OpenGL/ModernGL上下文创建失败 | 显示错误并提示更新显卡驱动；保留一个低保真 Pygame 2D 降级渲染作为可选兜底 |
| 音频设备不可用 | 静默降级，视觉不受影响，调试面板标记 `audio: off` |
| 帧率降到20fps以下 | dt增大→物理计算仍然准确。自动关闭Bloom，并将粒子物理更新降为隔帧或降低粒子数 |
| 粒子数到上限 | 轨迹粒子达上限→最旧的轨迹粒子加速衰减（寿命×0.5） |
| 窗口失焦/最小化 | 主循环继续低频运行或暂停。恢复后dt钳制到200ms。GA减半（模拟"气散了"） |
| 多人误入画面 | MediaPipe追踪最大/最稳定的人。如果第二个人干扰→追踪可能跳到第二个人。MVP不处理此情况，提示"请单人使用" |
| 用户用左手为主 | 左手力场强度0.6，右手0。如果右手不动左手动→力场仍然有效但弱。画面提示"推荐以右手为主"（调试模式可见） |
| Python依赖版本不兼容 | README 固定推荐版本；启动时打印 Python 版本、依赖导入结果和故障建议 |

## 十三、验收标准

**功能：**
- [ ] `python main.py --input mouse` 可启动窗口，鼠标可驱动右手力场
- [ ] `python main.py --input camera` 可启动摄像头，双手腕实时追踪
- [ ] 快速移动手/鼠标 → 粒子被推开，手周形成空洞
- [ ] 减速移动 → 粒子开始被吸引，身后形成轨迹尾
- [ ] 匀速慢行 → 轨迹流畅，粒子围绕手臂形成漩涡
- [ ] 停住3秒 → 粒子汇聚后爆发绽放
- [ ] 双手各自产生力场，鼠标模式下按住 `Shift` 可检查左手辅助力场，交汇时粒子桥形成
- [ ] GA累积→四阶金气依次呈现
- [ ] 不稳定→GA衰减，金气降阶
- [ ] 轨迹声音、激活声音、绽放和弦、金气风铃
- [ ] D键调试面板

**性能：**
- [ ] 稳定30fps以上（6000粒子+关闭Bloom）
- [ ] 接近60fps（6000-8000粒子+低强度Bloom，现代硬件）
- [ ] 10分钟连续运行不卡顿、不崩溃、内存稳定
- [ ] 摄像头不可用时自动进入鼠标模式，不影响视觉验收

**体验：**
- [ ] 不懂太极的人30秒内自己发现"慢=粒子跟你走"
- [ ] 站桩稳定触发绽放
- [ ] 画面在暗色屏幕上具有视觉冲击力（适合展览）
- [ ] 粒子运动有机感强，不像"程序生成的"
