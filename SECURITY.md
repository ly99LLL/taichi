# 安全政策

## 支持范围

安全修复仅面向 `master` 分支的最新代码。历史标签仅用于归档，不保证继续维护。

## 报告漏洞

请不要为未公开漏洞创建公开 Issue。优先使用 GitHub 仓库的
**Security → Report a vulnerability** 私密报告功能，并提供：

- 受影响版本或提交
- 复现步骤与最小样例
- 潜在影响
- 已知的缓解方式（如有）

维护者会尽量在 7 天内确认收到报告，并在验证后协调修复与披露时间。若仓库尚未启用
私密漏洞报告，请通过 GitHub 个人主页联系维护者，并只发送不含利用细节的初始说明。

本项目会处理摄像头画面。画面默认只在本机实时处理；贡献新功能时，不得在未经用户明确
同意的情况下上传、持久化或遥测图像数据。
