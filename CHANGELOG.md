# 更新日志

本项目遵循[语义化版本](https://semver.org/lang/zh-CN/)，从 v3.0.1 起在此记录重要变化。

## [未发布]

### 新增

- 双手涡场生命周期：成核、保持、解旋、余涡与重新成核。
- 基于 handedness 与位置预测的稳定双手身份槽。
- 双手靠近时的反向旋涡 `∞` 形流桥。
- `VortexController` 的纯 CPU 生命周期测试。
- Python 包元数据、开发依赖和统一工具配置。
- Python 3.11/3.12 CPU 测试、Ruff 与覆盖率 GitHub Actions。
- 可选 CUDA 集成测试、pre-commit 和 Dependabot。
- 许可证、贡献指南、安全政策、行为准则和 Issue/PR 模板。
- 视频输入与主窗口录制能力。
- 可关闭右下角视频预览的离线渲染模式。
- 不读取摄像头或外部媒体的确定性合成演示生成器。
- 真实视频与合成演示共用的 OpenCV 粒子帧渲染器。
- 无人物、无摄像头画面的 README 演示封面。
- Apple Silicon 使用不依赖芯片代际的 arm64 / Metal 兼容规则，覆盖 M1–M5 及后续机型。
- 实时入口和全部离线脚本共用 `auto/cpu/cuda/metal/vulkan/opengl` 后端选项。

### 修复

- 检测列表换序不再导致双手涡旋身份瞬间互换。
- 长时间丢失后重新检测不会把位置跳变误算成巨幅手速。
- 使用真实 MediaPipe Z 坐标计算纵深速度，不再误用屏幕 Y 坐标。
- 启动时尊重现有 `JAVA_HOME`，移除作者电脑上的 JDK 绝对路径。
- 修正 `py5` 预发行版本约束，避免全新环境无法解析依赖。
- 启动时检测 Python 与 Java 的处理器架构，避免 Apple Silicon 混用 arm64 与 Rosetta JDK。

### 变更

- 粒子总量提高到 7200，基础粒径略增，空间尘埃密度提升约五分之一。
- 缩短新涡旋成核时间，并提高涡心位置响应、速度继承和轨道约束，慢速移动时更跟手、更快汇聚。
- 新涡旋成核时先在掌心形成小核，再以不规则半径扩散到涡圈边缘。
- 新增急停惯性泼洒：手部快速移动后突然停下时，粒子会短暂继承动量并沿运动方向甩出。
- 新增高速丢检短时预测补位，右下角相机小窗会显示运动记忆光圈，减少快手时识别光效瞬间消失。
- Pose 腕关节降级补位现在只在 Hands 已识别到一只手时补齐第二只；Hands 全丢时不强行补位，重叠手不再按距离去重。
- 当 Hands 只识别到一只手时，会用 Pose 左/右腕位置反校准该手的左右标签，再补缺失的另一只，降低交叉动作时只剩单手的概率。
- 双手涡圈接触区由中轴吸附改为弹性排斥与擦边滑移，减轻两圈相遇时被压扁的视觉。
- 提高涡心位置与速度响应，增强慢速移动时的粒子携带和轨道约束。
- 提前进入快速解旋区间，确保提高跟手性后高速动作仍会明确发散。
- 粒子物理由“手掌六种扰动力”完全替换为常驻尘场与空心轨道动力学。
- 无手时粒子回归低亮度背景尘场，不再瞬间失去作用场。
- 视觉改为电影黑场、冷银尘埃、旧金相干涡旋和冷蓝破碎态。
- 摄像头小窗保留原彩画质，增加左右手双色骨架、指尖光点和掌心断环。
- 移除全画面冷灰滤镜、掌心大面积光晕与多层装饰边框。
- 移除仓库中含第三方人物画面的旧预览图，根目录临时预览默认不再纳入版本控制。
- README 按公开项目规范补全安装、运行模式、架构、测试、隐私和限制说明。
- README 精简兼容性与使用说明，并明确 M5 支持依据；本地 `presentation/` 展示网页默认忽略。
